# Plan 63-02: Keyword Universe Builder + Integration

## Goal
Generate 100-200 on-point keywords per prospect using DataForSEO autocomplete, existing keyword APIs, and the classification cascade. Integrate with confirmation UI for human review.

## Context
- **Gap G5:** No autocomplete API integration
- **Gap G8:** No human_confirmation setting
- **Gap G10:** SSE progress doesn't show classification stage

## Tasks

### Task 1: Add DataForSEO Autocomplete Wrapper (2h)

**File:** `open-seo-main/src/server/lib/dataforseo.ts`

Add fetchAutocomplete with cursor pointer variation (5 positions per seed).

### Task 2: Create KeywordUniverseBuilder Service (4h)

**File:** `open-seo-main/src/server/features/keywords/services/KeywordUniverseBuilder.ts`

Orchestrate: autocomplete → suggestions → ideas → PAA → deduplicate → classify.

### Task 3: Create Classification Cascade Orchestrator (2h)

**File:** `open-seo-main/src/server/features/keywords/services/classification/ClassificationCascade.ts`

Pass 1 (Grok 4.1) → Pass 2 (Claude Sonnet) → Human Review queue.

### Task 4: Add Human Confirmation Setting (1.5h)

**Files:** `apps/web/src/stores/prospect-wizard-store.ts`, `apps/web/src/components/prospects/ConfirmationSettings.tsx`

Add keywordConfirmation: 'always' | 'never' | 'low_confidence'.

### Task 5: Wire Classification to Confirmation Flow (2h)

**Files:** `apps/web/src/app/(shell)/prospects/actions.ts`, `apps/web/src/components/prospects/ExtractionConfirmation.tsx`

Show classified keywords with type badges, include/exclude toggles.

### Task 6: Add Classification Stage to SSE Progress (0.5h)

**File:** `apps/web/src/hooks/useAnalysisProgress.ts`

Add 'discovering_keywords' and 'classifying_keywords' stages.

### Task 7: Integration Tests (2h)

E2E tests for Lithuanian keywords, negative association filtering, cost tracking.
