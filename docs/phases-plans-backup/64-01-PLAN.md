# Plan 64-01: Crawl Singleflight

## Goal
Prevent 98% cost waste from duplicate crawl requests using Redis `SET key NX EX` singleflight pattern.

## Context
- **Current:** Multiple jobs can crawl same URL simultaneously
- **Problem:** 98% wasted API/compute cost on duplicates
- **Solution:** Redis singleflight coalescence
- **Reference:** `docs/infra-research/crawling-10-5000-tasks-day.md`

## Tasks

### Task 1: Create CrawlSingleflight Service (3h)

**File:** `open-seo-main/src/server/features/crawl/services/CrawlSingleflight.ts`

Lock via `SET key NX EX ttl`, cache result, waiter polling, timeout handling.

### Task 2: Create URL Normalization (1.5h)

**File:** `open-seo-main/src/server/features/crawl/utils/normalizeUrl.ts`

Remove trailing slashes, sort query params, filter tracking params, lowercase hostname.

### Task 3: Integrate with CrawlService (2.5h)

Wrap all crawl requests through singleflight layer.

### Task 4: Add Singleflight Metrics (1.5h)

Track executed/waited/cached ratio and estimated savings.

### Task 5: Unit Tests (1.5h)

Test concurrency, caching, timeout scenarios.

## Verification

```bash
pnpm --filter open-seo-main test CrawlSingleflight
# Integration: 10 concurrent crawls → only 1 actual execution
```
