# Plan 65-01: FalkorDB Schema + Per-Tenant Graphs

## Goal
Implement FalkorDB graph database with per-workspace isolation for knowledge graphs.

## Context
- **Reference:** `docs/infra-research/cpu-only-rag-graph.md`
- **Target:** $50/mo VPS compatible (CPU-only)

## Graph Schema

```cypher
(:Prospect {id, name, domain, industry, workspaceId})
(:Keyword {id, keyword, normalized, volume, difficulty, workspaceId})
(:Page {id, url, title, wordCount, workspaceId})
(:Topic {id, name, workspaceId})

(:Prospect)-[:HAS_KEYWORD]->(:Keyword)
(:Keyword)-[:RANKS_ON]->(:Page)
(:Keyword)-[:BELONGS_TO]->(:Topic)
(:Page)-[:LINKS_TO]->(:Page)
(:Topic)-[:SUBTOPIC_OF]->(:Topic)
```

## Tasks

### Task 1: FalkorDB Setup (3h)

Docker compose addition, connection pooling, health checks.

### Task 2: Graph Schema Implementation (4h)

Node types, relationship types, indexes, constraints.

### Task 3: Per-Tenant Isolation (3h)

Workspace-scoped queries, tenant filtering on all operations.

### Task 4: Repository Layer (2h)

GraphRepository with CRUD operations for all node types.
