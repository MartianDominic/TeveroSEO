# Plan 63-03: Human Confirmation Toggle + Adaptive Intent

## Goal
Add confirm/autonomous toggle for human-in-the-loop control. Make the system "Claude Code-like" by adapting tool selection based on detected intent.

## Context
- **User Request:** "add confirm/autonomous toggle near input for human-in-the-loop control"
- **Gap:** System runs fixed 5-step pipeline regardless of user intent
- **Audit:** 56-AUDIT-PROSPECT-FLOW.md rated system 5/10 - "FIXED PIPELINE not intelligent agent"

## Tasks

### Task 1: Create Confirmation Toggle Component (2h)

**File:** `apps/web/src/components/prospects/ConfirmationToggle.tsx`

Switch with localStorage persistence, tooltip explaining each mode.

### Task 2: Create Understanding Confirmation Dialog (3h)

**File:** `apps/web/src/components/prospects/UnderstandingConfirmation.tsx`

Shows extracted fields, click-to-edit, confidence indicator, confirm/cancel actions.

### Task 3: Add Intent Detection (2.5h)

**File:** `open-seo-main/src/server/features/prospects/services/IntentDetector.ts`

Detect quick_check, competitor_analysis, keyword_discovery, full_analysis, update_existing.

### Task 4: Wire Confirmation to Prospect Flow (2.5h)

**Files:** `apps/web/src/app/(shell)/prospects/actions.ts`, `apps/web/src/components/prospects/ProspectInput.tsx`

Return confirmation_required in confirm mode, proceed directly in autonomous mode.

### Task 5: Update SSE Progress with Intent (1h)

**File:** `apps/web/src/hooks/useAnalysisProgress.ts`

Add 'understanding' and 'confirming' stages, show detected intent.

### Task 6: Unit Tests (1h)

Test toggle persistence, dialog fields, intent detection accuracy, edit flow.
