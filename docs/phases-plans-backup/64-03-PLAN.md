# Plan 64-03: Fast/Heavy Queue Lanes

## Goal
Implement dual queue lanes with different SLAs.

## Task Shapes

| Type | Description | Lane | SLA |
|------|-------------|------|-----|
| A | Full audit | heavy | 15m |
| B | Delta recheck | fast | 1m |
| C | Pure API (DataForSEO) | fast | 30s |
| D | SERP-derived | fast | 1m |
| E | Content analysis | fast | 2m |
| F | Keyword-driven | fast | 2m |

## Tasks

### Task 1: Create Queue Configuration (2h)

Define BullMQ queues with separate concurrency and rate limits.

### Task 2: Create TaskRouter Service (2.5h)

Route tasks to appropriate lane based on task shape detection.

### Task 3: Update Job Processors (2h)

Wire processors to correct queues with lane-specific settings.

### Task 4: Add Lane Metrics (1.5h)

Track throughput, latency, and SLA compliance per lane.

## Verification

```bash
# Quick check task completes in <30s (fast lane)
# Full audit task uses heavy lane, doesn't block fast tasks
```
