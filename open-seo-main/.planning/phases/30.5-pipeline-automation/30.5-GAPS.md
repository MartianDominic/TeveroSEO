## Gap 1: Prospect Automation Cron Not Wired Up (CRITICAL)

**Status:** Code exists but NOT connected to execution

**Problem:** The `processProspectAutomations()` function exists but is never called from any cron endpoint or scheduler.

**Evidence:**
- `src/routes/api/cron/automations.ts` only calls `processAutomations()` (proposals), not `processProspectAutomations()` (prospects)
- grep for `processProspectAutomations` only shows:
  - Definition in `src/server/features/prospects/automation/prospectAutomation.ts:123`
  - Tests in `prospectAutomation.test.ts`
  - **No cron/scheduler integration**

**Files to fix:**
- `src/routes/api/cron/automations.ts` - Add call to `processProspectAutomations`
  
OR create dedicated endpoint:
- Create `src/routes/api/cron/prospect-automations.ts`

**Impact:** Auto-qualify (score >= 70) rules will NEVER run via cron. They only work via `PipelineService.handleAnalysisComplete()` which is called after analysis completes.
