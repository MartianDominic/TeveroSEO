  ---
  The Core Problem

  You have 5 different input sources, but AI generation needs to work the 
  same way regardless of source:

  ┌────────────────────────────┬────────────────────────────────────────┐
  │           Source           │               Challenge                │
  ├────────────────────────────┼────────────────────────────────────────┤
  │ Copy-paste from Google Doc │ Raw text, no structure                 │
  ├────────────────────────────┼────────────────────────────────────────┤
  │ Design from scratch        │ Empty blocks need prompts              │
  ├────────────────────────────┼────────────────────────────────────────┤
  │ Templates from pasting     │ Need to mark what's fixed vs dynamic   │
  ├────────────────────────────┼────────────────────────────────────────┤
  │ Templates built locally    │ Same as above, different creation path │
  ├────────────────────────────┼────────────────────────────────────────┤
  │ PDF upload                 │ No semantic structure, may have images │
  └────────────────────────────┴────────────────────────────────────────┘

  ---
  World-Class Architecture: 3-Layer System

  ┌─────────────────────────────────────────────────────────────┐
  │  LAYER 1: STRUCTURE                                         │
  │  (Persuasion framework — block types + order)               │
  │  "This is a Perfect Webinar proposal with 8 blocks"         │
  └─────────────────────────────────────────────────────────────┘
                                ↓
  ┌─────────────────────────────────────────────────────────────┐
  │  LAYER 2: CONTENT                                           │
  │  (Actual text — fixed, variable, or AI-generated)           │
  │  "Mes sutartyje įsipareigojame per 6 mėnesius..."          │
  └─────────────────────────────────────────────────────────────┘
                                ↓
  ┌─────────────────────────────────────────────────────────────┐
  │  LAYER 3: CONTEXT                                           │
  │  (Prospect data + style references for AI)                  │
  │  "SEO agency, Lithuanian market, plaukupasaka.lt"          │
  └─────────────────────────────────────────────────────────────┘

  This separation means:
  - Any source can be converted to Structure + Content
  - AI generation always receives the same input format
  - Templates define Structure + Content, with markers for what AI should
  regenerate

  Recommended: PDF as Style Reference

  Upload PDF
       ↓
  Stored as "style reference" (not editable template)
       ↓

  "Write a Risk Reversal block in the style of this PDF"

  Why? PDF editing is a nightmare. Better to use PDFs as style guides, not

  ---
  3. Templates (from paste OR built locally)
  
  Templates need content modes per block:

  ┌────────────┬──────────────────────────┬────────────────────────────┐
  │    Mode    │         Meaning          │          Example           │
  ├────────────┼──────────────────────────┼────────────────────────────┤
  │ FIXED      │ Same text every time     │ Your guarantee language    │
  ├────────────┼──────────────────────────┼────────────────────────────┤
  │ VARIABLE   │ Has {{ }} placeholders   │ "Dear {{ prospect.name }}" │
  ├────────────┼──────────────────────────┼────────────────────────────┤
  │ REGENERATE │ AI generates fresh each  │ Pain point specific to     │
  │            │ time                     │ prospect                   │
  └────────────┴──────────────────────────┴────────────────────────────┘

  interface TemplateBlock {
    persuasionType: PersuasionType;
    contentMode: 'fixed' | 'variable' | 'regenerate';

    // If fixed:
    fixedContent?: string;

    // If variable:
    variableTemplate?: string;  // "Dear {{ prospect.name }}"

    // If regenerate:
    regenerationPrompt?: string;  // "Write about their SEO pain points"
    styleReferenceContent?: string;  // Original text as style guide
  }

  Creating a template from paste:
  Paste proposal → AI segments → User confirms structure
           ↓
  For each block, user marks:
    ☑ FIXED - "Keep this exact text" (guarantee language)
    ☐ VARIABLE - "Add placeholders" (prospect name, industry)
    ☐ REGENERATE - "AI writes fresh each time, use this as style guide"
           ↓
  Template saved with structure + content modes

  ---
  AI Generation: Unified Flow

  Regardless of source, AI generation always receives:

  interface GenerationRequest {
    // Layer 1: What type of block
    blockType: PersuasionType;
    blockPurpose: string;  // "Your guarantee - what if it doesn't work?"

    // Layer 2: Prospect context (always included)
    prospect: {
      domain: string;
      industry: string;
      painPoints: string[];  // From SEO analysis
      language: 'en' | 'lt';
    };

    // Layer 3: Style references (when available)
    styleGuides?: {
      templateContent?: string;  // If regenerating from template
      pdfReferences?: string[];  // Extracted text or image URLs
      priorWinningBlocks?: string[];  // Blocks with high close rates
    };

    // Layer 4: Coherence (smart, not expensive)
    priorBlocksSummary?: string;  // 3-sentence AI summary, NOT full text

    // Layer 5: Constraints
    constraints: {
      tone: string;  // "Lithuanian, direct-response"
      lengthRange: [number, number];  // [300, 500] words
      brandVoice?: string;  // Client's voice profile if defined
    };
  }

  Cost analysis:
  - Block purpose + prospect context: ~200 tokens
  - Style reference (if included): ~500 tokens
  - Prior blocks summary: ~100 tokens
  - Output: ~500 tokens
  - Total: ~$0.002 per block with Gemini 3.1 Pro
