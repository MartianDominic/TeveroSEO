# Phase 16 Plan 04: Branding Settings UI Summary

White-label branding UI with live preview and report template branding injection.

## One-Liner

Branding settings UI with logo upload, color pickers, live preview, and report template injection using hexToRgb for PDF compatibility.

## What Was Built

### 1. Branding Settings Page (`/clients/[id]/settings/branding`)

- **LogoUpload.tsx:** Drag-and-drop file upload with preview
  - Validates PNG/JPG/SVG, max 2MB
  - Shows current logo with replace/delete options
  - Visual feedback during upload

- **ColorPicker.tsx:** Hex color input with visual picker
  - Native color picker + manual hex input
  - Preset color swatches for quick selection
  - Live validation of hex format

- **BrandingPreview.tsx:** Live preview of branded report header
  - Shows logo, primary color, client name
  - Updates in real-time as settings change
  - Accurate representation of PDF output

- **BrandingForm.tsx:** Orchestrates all components
  - Save/reset functionality with toast feedback
  - Tracks unsaved changes
  - Integrates with branding API

### 2. Schedule Settings Page (`/clients/[id]/settings/reports`)

- **ScheduleForm.tsx:** Weekly and monthly schedule configuration
  - User-friendly day/time selectors (not raw cron)
  - Timezone selection with auto-detect
  - Recipients email input with validation
  - Next delivery preview showing calculated date/time
  - Enable/disable toggles per schedule type

### 3. Report Template Branding Injection

- **ReportHeader.tsx:** Added optional `branding` prop
  - Displays logo when `branding.logoUrl` is set
  - Uses `branding.primaryColor` for heading (with hexToRgb conversion)
  - Falls back to Tevero defaults when not branded

- **ReportFooter.tsx:** Added optional `branding` prop
  - Uses `branding.footerText` when set (pre-sanitized HTML via DOMPurify in API)
  - Falls back to "Generated by Tevero" when not branded

- **report-renderer.ts (open-seo-main):** Server-side HTML generation
  - Added `ReportBranding` interface
  - `renderHeader` and `renderFooter` accept branding parameter
  - `renderReportToHTML` passes branding through

- **report-processor.ts (open-seo-main):** Fetches branding before render
  - Queries `db.query.clientBranding` with clientId
  - Passes branding to `renderReportToHTML`
  - Logs branding status for debugging

### 4. API Client Functions

- **branding.ts:** `getBranding`, `updateBranding`, `uploadLogo`, `deleteLogo`, `resetBranding`
- **schedules.ts:** `getSchedules`, `createSchedule`, `updateSchedule`, `deleteSchedule`
  - Helper functions: `buildCronExpression`, `parseCronExpression`, `calculateNextRun`
  - Constants: `COMMON_TIMEZONES`, `DAYS_OF_WEEK`, `DAYS_OF_MONTH`, `HOURS`
