| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Spoofing | Schedule API | mitigate | Validate clientId ownership via Clerk org membership before CRUD |
| T-16-02 | Tampering | Cron expression | mitigate | Validate cron syntax with cron-parser; reject invalid; limit to predefined patterns if needed |
| T-16-03 | Repudiation | Schedule changes | accept | No audit log for v1; low-value target |
| T-16-04 | Information Disclosure | Recipient emails | mitigate | Only return schedules for authenticated client; no cross-client access |
| T-16-05 | Denial of Service | Frequent schedules | mitigate | Minimum interval validation (e.g., no more than daily); rate limit API |
| T-16-06 | Elevation of Privilege | Schedule for other client | mitigate | Strict clientId validation against authenticated user's org |
</threat_model>
