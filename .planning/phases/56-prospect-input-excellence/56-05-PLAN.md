# Plan 56-05: Business Context Extraction + Classification Foundation

## Goal
Enable contextual keyword filtering by extracting what businesses DON'T do (negative associations) and their business model (B2B/B2C/ecommerce), then build the multi-tier classification cascade infrastructure from 56-CASCADE-ARCHITECTURE.md.

## Context
- **Gap G1:** No negative association extraction (can't filter adjacent verticals)
- **Gap G2:** No business model detection (can't filter by B2B/B2C intent)
- **Gap G3:** No binary relevance classification (need 0.85 confidence gate)
- **Reference:** 56-AUDIT.md, 56-CASCADE-ARCHITECTURE.md, 56-RESEARCH-MODEL-COSTS.md

## Tasks

### Task 1: Extend ConversationExtractor Prompt (2h)

**File:** `open-seo-main/src/server/features/prospects/prompts/conversation-extractor.xml`

Add new extraction blocks:

```xml

  <not_services>Things they explicitly DO NOT sell/provide</not_services>
  <competitors>Known competitor domains or business types</competitors>
  <adjacent_verticals>Related but contextually wrong verticals</adjacent_verticals>

</business_model>
```

  name: z.string(),

]);

  keyword: z.string(),
  classification: z.enum(["clearly_include", "clearly_exclude", "uncertain"]),
  confidence: z.number().min(0).max(1),

});

export const ClassifiedKeywordSchema = z.object({
  keyword: z.string(),
  normalizedKeyword: z.string(),
  include: z.boolean(),
