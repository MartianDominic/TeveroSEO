# Phase 18 Plan 01: Alert Schema + Rule Engine

## Objective

Create Drizzle schema for alert rules and alerts. Build core alert service for creating, querying, and managing alerts.

## Tasks

### Task 1: Create alert schema

**Files:**
- `open-seo-main/src/db/alert-schema.ts`
- `open-seo-main/src/db/schema.ts`

**Changes:**
1. Create `alertRules` table:
   - `id` (text, UUID)
   - `clientId` (text)
   - `alertType` (text: ranking_drop, sync_failure, connection_expiry)
   - `enabled` (boolean, default true)
   - `threshold` (integer, nullable)
   - `severity` (text: info, warning, critical)
   - `emailNotify` (boolean, default false)
   - `createdAt`, `updatedAt` (timestamps)
   - Unique index on (clientId, alertType)

2. Create `alerts` table:
   - `id` (text, UUID)
   - `clientId` (text)
   - `ruleId` (text, nullable, FK to alertRules)
   - `alertType` (text)
   - `severity` (text)
   - `status` (text: pending, acknowledged, resolved, dismissed)
   - `title` (text)
   - `message` (text)
   - `metadata` (jsonb)
   - `sourceEventId` (text, nullable) — links to rank_drop_events.id
   - `createdAt`, `acknowledgedAt`, `resolvedAt` (timestamps)
   - Indexes on (clientId, status), (clientId, createdAt)

3. Export from schema.ts barrel

**Commit:** `feat(18-01): add alert schema`

### Task 2: Create alert service

**Files:**
- `open-seo-main/src/services/alerts.ts`

**Changes:**
1. `createAlert(data)` — insert new alert
2. `getClientAlerts(clientId, filters)` — list alerts with optional status filter
3. `getUnacknowledgedCount(clientId)` — for badge count
4. `acknowledgeAlert(id)` — set status to acknowledged
5. `resolveAlert(id)` — set status to resolved
6. `dismissAlert(id)` — set status to dismissed
7. `getAlertRule(clientId, alertType)` — get rule for processing
8. `upsertAlertRule(data)` — create or update rule

**Commit:** `feat(18-01): add alert service`
