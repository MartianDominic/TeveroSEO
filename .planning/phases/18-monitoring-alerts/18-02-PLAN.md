# Phase 18 Plan 02: Alert Creation in Workers

## Objective

Create BullMQ alert processor that consumes rank_drop_events and creates alerts. Integrate with ranking and analytics workers to emit alert-worthy events.

## Tasks

### Task 1: Create alert queue definition

**Files:**
- `open-seo-main/src/server/queues/alertQueue.ts`

**Changes:**
1. Define `ALERT_QUEUE_NAME` = "alert-processing"
2. Define job types: process_drop_events, check_sync_failures, check_connection_expiry
3. Create `initAlertScheduler()` for repeatable jobs (every 5 minutes)
4. Export queue instance and job type definitions

**Commit:** `feat(18-02): add alert queue`

---

### Task 2: Create alert processor

**Files:**
- `open-seo-main/src/server/workers/alert-processor.ts`

**Changes:**
1. Process `process_drop_events` job type:
   - Query unprocessed rank_drop_events
   - For each event, check client's alert rule
   - Create alert with appropriate severity
   - Mark event as processed
   - Return list of created alerts (for notification service)

2. Process `check_sync_failures` job type:
   - Query recent sync failures from analytics queue
   - Create alerts for persistent failures

3. Job data includes `alertsCreated` array for chaining to notification

**Commit:** `feat(18-02): add alert processor`

---

### Task 3: Create alert worker

**Files:**
- `open-seo-main/src/server/workers/alert-worker.ts`
- `open-seo-main/src/worker-entry.ts`

**Changes:**
1. Create `startAlertWorker()` and `stopAlertWorker()`
2. Configure worker:
   - lockDuration: 60_000 (1 minute)
   - concurrency: 1
   - maxStalledCount: 2
3. Add to worker-entry.ts startup/shutdown

**Commit:** `feat(18-02): add alert worker`
