# Phase 18 Plan 03: Alert Notification Service

## Objective

Send email notifications for high-severity alerts using existing Resend integration. Track notification status on alerts.

## Tasks

### Task 1: Add notification fields to alerts schema

**Files:**
- `open-seo-main/src/db/alert-schema.ts`
- `open-seo-main/drizzle/0007_alerts.sql`

**Changes:**
1. Add `emailSentAt` timestamp column to alerts table
2. Add `emailError` text column for failed sends
3. Update migration to include columns

**Commit:** `feat(18-03): add notification fields to alerts`

---

### Task 2: Create alert notification service

**Files:**
- `open-seo-main/src/services/alert-notifications.ts`

**Changes:**
1. `shouldSendEmail(alert, rule)` — check if email should be sent
2. `sendAlertEmail(alert, clientEmail)` — send via Resend
3. `getAlertEmailTemplate(alert)` — generate email content
4. Email includes: alert title, message, severity badge, link to dashboard

**Commit:** `feat(18-03): add alert notification service`

---

### Task 3: Integrate notifications with alert processor

**Files:**
- `open-seo-main/src/server/workers/alert-processor.ts`

**Changes:**
1. After creating alert, check if email notification enabled
2. Call notification service for qualifying alerts
3. Update alert with emailSentAt or emailError
4. Log notification outcomes

**Commit:** `feat(18-03): integrate notifications in alert processor`
