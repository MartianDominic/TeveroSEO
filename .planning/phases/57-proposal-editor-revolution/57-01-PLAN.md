# Plan 57-01: Schema + Template CRUD + i18n Setup

## Goal
Establish the template system foundation with database schema, CRUD service, API endpoints, and i18n infrastructure.

## Tasks

### Task 1: Drizzle Schema (2h)

Create `proposal_templates` table:
- `id`, `workspaceId` (null = system template)
- `name`, `nameEn`, `nameLt`, `description` (localized)
- `type` ('proposal' | 'case_study' | 'report')
- `category` ('seo', 'local_seo', 'ecommerce', 'enterprise')
- `sections` (jsonb), `sectionOrder` (jsonb), `variables` (jsonb)
- `brandingSettings` (jsonb)
- `version`, `isPublished`, `isDefault`
- `createdAt`, `updatedAt`, `createdBy`

Create `template_sections` table:
- `id`, `templateId` (FK)
- `key`, `title`, `titleEn`, `titleLt`
- `content`, `contentEn`, `contentLt`
- `sectionType`, `isRequired`, `isEditable`, `position`
- `conditions` (jsonb), `aiPromptHint`

### Task 2: Template Repository (1.5h)

Repository pattern with:
- `findAll(workspaceId)` — list templates for workspace + system
- `findById(id)` — get template with sections
- `create(data)` — create template
- `update(id, data)` — update template
- `delete(id)` — soft delete template
- `duplicate(id, workspaceId)` — clone system to workspace

### Task 3: TemplateService (2h)

Business logic layer:
- Template validation
- Default template selection
- System template seeding
- Workspace template inheritance

### Task 4: API Endpoints (1.5h)

```
GET    /api/templates/proposals          # List for workspace
GET    /api/templates/proposals/:id      # Get with sections
POST   /api/templates/proposals          # Create
PUT    /api/templates/proposals/:id      # Update
DELETE /api/templates/proposals/:id      # Delete
POST   /api/templates/proposals/:id/duplicate  # Clone
```

### Task 5: i18n Setup (1h)

Create translation files:
- `apps/web/src/locales/en/proposal-editor.json`
- `apps/web/src/locales/lt/proposal-editor.json`

Include editor, sections, variables, AI, and saveStatus keys.
