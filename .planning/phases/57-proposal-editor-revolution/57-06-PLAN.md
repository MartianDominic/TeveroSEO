# Plan 57-06: Auto-Save + Version History

## Goal
Implement automatic saving with debounce and version history with restore.

## Dependencies

```bash
pnpm add use-debounce
```

## Tasks

### Task 1: useAutoSave Hook (2h)

```typescript
function useAutoSave(proposalId: string, content: ProposalContent) {
  const [saveStatus, setSaveStatus] = useState<'saved' | 'saving' | 'error'>('saved');
  const [lastSavedAt, setLastSavedAt] = useState<Date | null>(null);

  const debouncedSave = useDebouncedCallback(
    async (content) => {
      setSaveStatus('saving');
      try {
        await saveProposal(proposalId, content);
        setSaveStatus('saved');
        setLastSavedAt(new Date());
      } catch {
        setSaveStatus('error');
        queueOfflineSave(proposalId, content);
      }
    },
    2000
  );

  useEffect(() => {
    debouncedSave(content);
  }, [content]);

  return { saveStatus, lastSavedAt };
}
```

### Task 2: SaveIndicator Component (1h)

Status display:
- "Saving..." with spinner
- "Saved" with checkmark + relative time
- "Save failed" with warning + retry button
- All labels localized (EN/LT)

### Task 3: Version Schema + Service (3.5h)

Create `proposal_versions` table:
- `id`, `proposalId` (FK with cascade)
- `content` (jsonb), `sectionOrder` (jsonb)
- `versionNumber` (auto-increment per proposal)
- `changeDescription`, `changeDescriptionEn`, `changeDescriptionLt`
- `changeType` (content_edit, section_reorder, section_add, ai_generated)
- `changedSections` (string[])
- `createdAt`, `createdBy`

VersionService:
- `createVersion(proposalId, content, changeType)` — snapshot current state
- `listVersions(proposalId)` — list all versions
- `getVersion(versionId)` — get specific version
- `restoreVersion(versionId)` — overwrite current with version
- Auto-version on significant changes (not every save)

### Task 4: VersionHistory Component + API (2.5h)

Sidebar panel:
- List of versions with date, author, change description
- Current version highlighted
- Preview button shows version content
- Restore button with confirmation
- Relative timestamps

API Endpoints:
```
GET    /api/proposals/:id/versions            # List versions
GET    /api/proposals/:id/versions/:vid       # Get version
POST   /api/proposals/:id/versions/:vid/restore  # Restore version
```
