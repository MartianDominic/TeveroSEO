Create target-selector.ts per design doc:

1. rankLinkTargets(sourceUrl, clientId, auditId, limit = 10)
   ```typescript
   interface TargetCandidate {
     pageUrl: string;
     pageId: string;
     title: string;
     keyword: string | null;
     inboundLinks: number;
     score: number;
     reasons: string[];
   }
   ```
   - Get all pages from page_links (exclude source)
   - Get source page content for relevance
   - Get existing links from source (via link_graph)
   - Build alreadyLinked Set
   - Score each candidate:
     - Factor 1: Link deficit score (0.25 weight) - linksNeeded / 40
     - Factor 2: No exact-match anchor (+0.2) - if inboundExactMatch === 0
     - Factor 3: Is orphan page (+0.3) - if inboundTotal === 0
     - Factor 4: High click depth (+0.15) - if clickDepthFromHome > 3
     - Factor 5: Topical relevance (0.2 weight) - keyword overlap
   - Filter out already-linked targets
   - Sort by score descending
   - Return top N with reasons

2. extractKeywordsFromContent(content: string): Set<string>
   - Stopword list (the, a, an, is, are, was, were, be, etc.)
   - Tokenize and lowercase
   - Filter out stopwords and words < 3 chars
   - Extract bigrams (consecutive non-stopwords)
   - Count frequency
   - Return top 50 by frequency

3. computeKeywordOverlap(set1: Set<string>, set2: Set<string>): number
   - Calculate intersection count
   - Calculate union count
   - Return Jaccard similarity: intersection / union
   - Handle empty sets (return 0)
  </action>

Create anchor-selector.ts per design doc:

1. selectAnchorText(options: AnchorTextOptions): AnchorTextResult
   ```typescript
   interface AnchorTextOptions {
     targetUrl: string;
     targetPageId: string;
     targetKeyword: string | null;
     targetTitle: string;
     brandName: string;
     currentAnchorDistribution: {
       exactMatch: number;
       branded: number;
       url: number;
       misc: number;
     };
     sourceContent: string;
   }

   interface AnchorTextResult {
     text: string;
     type: 'exact_match' | 'partial_match' | 'branded' | 'url' | 'misc';
     confidence: number;
     sourceText: string | null; // Existing text to wrap, or null if insertion needed
   }
   ```

   Logic:
   - Calculate total anchors and current ratios
   - Target ratios: exact=0.5, branded=0.25
   - Determine preferred type based on what's underrepresented:
     - If exactRatio < 0.5 && targetKeyword exists -> prefer exact_match
     - Else if brandedRatio < 0.25 -> prefer branded
     - Else -> misc
   - Check if preferred anchor exists in source content:
     - For exact_match: find targetKeyword in content (case-insensitive)
     - For branded: find brandName in content
     - If found: return with sourceText (wrap_existing), confidence 0.9-0.95
   - Fallback to title-based partial match:
     - Extract first 5 words of title
     - Check if exists in content
     - If found: return as partial_match, confidence 0.8
   - Last resort (no natural anchor):
     - Return targetKeyword or title words
     - sourceText: null (will need insertion)
     - confidence: 0.6

2. findTextInContent(needle: string, haystack: string): { found: boolean; exactText: string; index: number }
   - Case-insensitive search
   - If found, extract the exact case-preserved text from content
   - Return index for context extraction

Export from index.ts barrel.
