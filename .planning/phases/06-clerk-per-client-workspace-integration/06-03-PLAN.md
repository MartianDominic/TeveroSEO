<objective>
Wire the `X-Client-ID` header through the entire audit request path so the
platform honors per-client scoping end-to-end:

1. Extend `requireProjectContext` to resolve `clientId` once per request via
   `resolveClientId(headers)` and expose it on the server-function context.
2. Accept an optional `clientId` in the `getAuditHistorySchema` query input,
   validate it matches the header, and have `AuditService.getHistory` filter
   `audits` by `client_id`.
3. Have `AuditService.startAudit` persist `clientId` into the new column.
4. Document that authentication is already handled by `ensureUserMiddleware`
   (AUTH-01/AUTH-02) so the checker can sign off without rewrites.

Purpose: AUTH-01 (401 for unauthenticated) already works because
`ensureUserMiddleware` throws `UNAUTHENTICATED` when the better-auth session
is absent — this plan verifies + documents that. AUTH-02 (userId/orgId
available) is already true via `EnsuredUserContext`. AUTH-03/AUTH-04 are the
net-new wiring here.

Output:
- `requireProjectContext` middleware returns a context with `clientId: string | null`.
- `AuditService.startAudit({ clientId })` persists clientId on the audits row.
- `AuditService.getHistory(projectId, { clientId })` filters by client_id.
- `getAuditHistorySchema.clientId` added + 403 mismatch check added.
- Unit test covering the "schema clientId vs header clientId mismatch → FORBIDDEN" branch.
</objective>
