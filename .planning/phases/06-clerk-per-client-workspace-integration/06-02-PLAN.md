<objective>
[BLOCKING] Add a nullable `client_id` text column and a supporting index to
the `audits` table via a Drizzle schema edit + generated migration.

Purpose: AUTH-04 requires every audit row to carry its owning client so
`GET /api/audits?client_id=X` can filter server-side and so access checks
can compare header vs row. Existing audit rows predate per-client scoping
and must remain valid (hence nullable).

Output:
- `audits.clientId` column added in `src/db/app.schema.ts` (text, nullable).
- Composite index `audits_client_id_started_at_idx` on `(clientId, startedAt DESC)`.
- Fresh Drizzle migration file `drizzle/0001_audits_client_id.sql` generated via
  `drizzle-kit generate` (sits on top of existing `0000_init.sql`).
- `drizzle-kit migrate` applies cleanly against a fresh PostgreSQL instance.

**[BLOCKING for Wave 2]**: Plan 03 (AuditService client_id wiring) cannot run
until this migration exists and applies.
