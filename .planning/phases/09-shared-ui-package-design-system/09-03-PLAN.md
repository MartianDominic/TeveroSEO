B) `apps/web/next.config.ts` — add `transpilePackages`. The current file reads:

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  output: "standalone",
  reactStrictMode: true,
  typedRoutes: true,
};

```

Rewrite to:

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  output: "standalone",
  reactStrictMode: true,
  typedRoutes: true,
  transpilePackages: ["@tevero/ui", "@tevero/types"],
};

export default nextConfig;
```

Why `transpilePackages` (not a bundler in the package itself): packages/ui/package.json from plan 01 points `main` at `./src/index.ts` — raw TS. Next.js 15 will transpile those files during the apps/web build when the package name is listed here. No tsup, no tsc emit step, no watch mode. CONTEXT.md D-02 and the orchestrator constraint lock this decision in.

Do NOT remove the `"use client"` directive from any component file; Next.js handles client-component boundaries correctly inside transpiled packages (validated in Next 13.1+).
