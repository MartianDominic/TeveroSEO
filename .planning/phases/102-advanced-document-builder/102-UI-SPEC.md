## Interaction Patterns

### Drag-Drop Blocks

Following @dnd-kit patterns already in SectionHandle.tsx:

| State | Visual Treatment |
|-------|------------------|
| Draggable (at rest) | --shadow-card, cursor: grab on handle |
| Dragging | --shadow-lift, scale(1.02), opacity: 0.9 |
| Drop zone active | 4px border-dashed --accent, height: 8px expanding to 48px |
| Drop zone hover | border --accent-2, background --accent-soft |
| Invalid drop | cursor: not-allowed, drop zone red flash |

**Animation:**
- Drag pickup: 160ms var(--ease-quick)
- Drop settle: 280ms var(--ease-smooth)
- Reorder shift: 240ms var(--ease-smooth)

### Hover-to-Reveal Actions

Per Design System v6 section 11:

| Element | Resting | On Hover |
|---------|---------|----------|
| Block actions (edit, AI, variant, delete) | opacity: 0 | fade in 240ms |
| Drag handle | opacity: 0.5 | opacity: 1 |
| Delete button | hidden | fade in with red tint |
| AI sparkle icon | static | subtle pulse (if thinking disabled) |

### Block Selection

| State | Visual Treatment |
|-------|------------------|
| Unselected | --shadow-card |
| Hover | --shadow-lift, transform: translateY(-1px) |
| Selected | 2px solid --accent border, --shadow-card |
| Editing (TipTap active) | 2px solid --accent border, subtle glow |

### A/B Variant Tabs

| State | Visual Treatment |
|-------|------------------|
| Inactive variant | --surface-2 background, --text-3 |
| Active variant | --accent-soft background, --accent-ink text |
| Winner badge | --success-soft pill with checkmark |
| Loser badge | --error-soft pill with x |
| Needs data | --surface-3 pill, --text-4 |

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Keyboard drag-drop | Arrow keys to move, Enter to drop, Escape to cancel |
| Focus trap in modals | FrameworkSelector, VariantCreator |
| Screen reader announcements | "Block moved to position X", "Variant created" |
| Min touch target | 44px for all interactive elements |
| Color contrast | All text meets WCAG AA (4.5:1 body, 3:1 large) |
| Reduced motion | Honor prefers-reduced-motion (disable drag animations) |

## Motion Specifications

All motion per Design System v6 section 2.10:

| Interaction | Duration | Easing | Property |
|-------------|----------|--------|----------|
| Card hover lift | 280ms | var(--ease-smooth) | transform, box-shadow |
| Action reveal | 240ms | var(--ease-smooth) | opacity |
| Block reorder | 240ms | var(--ease-smooth) | transform |
| Modal open | 280ms | var(--ease-smooth) | opacity, transform |
| Skeleton shimmer | 1500ms | var(--ease-smooth) | background-position |
| Heatmap fade | 280ms | var(--ease-smooth) | opacity |

**Never:**
- Animations longer than 320ms
- Linear easing
- transform: scale() on hover (use translateY)
- Animate width/height of cards
