# 39-05: Production Verification Checklist

**Goal**: Final verification that the complete autonomous pipeline works end-to-end with GSC submission and link graph updates.

## Background

This is the final verification plan. All components exist:
- GSC URL submission via `submit_url_for_indexing()`
- Link graph update via `_run_link_graph_update()`
- Auto-publish cycle every 15 minutes via APScheduler

## Tasks

### Task 1: Verify GSC Service

```bash
# Check GSC service exists and handles auth
grep -n "submit_url_for_indexing" AI-Writer/backend/services/gsc_service.py

```

### Task 2: Verify Link Graph Update

```bash
# Check APScheduler job exists

```

### Task 4: E2E Test (Manual - Checkpoint Required)

**[HUMAN CHECKPOINT]** Run manual E2E test:

1. Create a scheduled article with:
   - `auto_publish = True`
   - `client_id` with valid CMS connection
   - `brief_context` with H2s and PAA questions

2. Trigger generation:
   ```bash
   curl -X POST http://localhost:8000/api/articles/{id}/generate
   ```

3. Verify generation flow:
   - [ ] Voice profile fetched from open-seo
   - [ ] H2 suggestions appear in prompt
   - [ ] PAA questions appear in prompt
   - [ ] Article generated successfully

4. Verify post-generation flow:
   - [ ] Internal links inserted (3-10 links)
   - [ ] Quality gate called
   - [ ] Article status = `approved` (if score >= 80)
   - [ ] OR Article status = `generated` (if score < 80)

5. Wait for publish cycle (or trigger manually):
   ```bash
   curl -X POST http://localhost:8000/api/publish/trigger
   ```

6. Verify publish flow:
   - [ ] Article published to CMS
   - [ ] GSC URL submission called (check logs)
   - [ ] Link graph update called (check logs)
   - [ ] Article status = `published`
