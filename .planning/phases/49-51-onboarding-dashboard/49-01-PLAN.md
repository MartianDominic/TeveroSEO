<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Magic link token → Backend | Untrusted token from URL path crosses into validation logic |
| OAuth callback → Completion service | External OAuth provider triggers internal state change |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-01 | Information Disclosure | Magic link token | mitigate | 32-char nanoid (128 bits entropy); rate limit validation endpoint; exclude token from logs |
| T-49-02 | Tampering | Magic link validation | mitigate | Token lookup only; no user-provided workspaceId/clientId trusted; all data from DB |
| T-49-03 | Denial of Service | Magic link generation | mitigate | Require authenticated workspace member to generate; 10 links/hour rate limit |
| T-49-04 | Elevation of Privilege | Auto-complete events | accept | Events dispatched from trusted OAuth callback routes only; no external API |
| T-49-05 | Repudiation | Checklist completion | mitigate | Activity log with entityType, actorId for all completions |
</threat_model>
