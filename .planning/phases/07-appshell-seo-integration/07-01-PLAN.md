<objective>
Extend `resolveClientId()` in open-seo-main so it reads `client_id` from the URL query
string when the `X-Client-ID` header is absent. This unblocks the cross-origin iframe
scenario (AI-Writer AppShell embeds `https://app.openseo.so/?client_id=<uuid>`) where
custom headers cannot be set from a parent frame.

Purpose: SHELL-04 requires `client_id` to flow from AI-Writer to open-seo on every
navigation. In a cross-origin iframe, URL query params are the only reliable channel.

Output: Updated `client-context.ts` with a new optional `url` argument, updated call
sites in `middleware.ts` that pass the request URL, and updated tests covering the
new code path.
</objective>

    Update signature and body of `resolveClientId`:

    ```typescript
    export const CLIENT_ID_QUERY_PARAM = "client_id";

    export async function resolveClientId(
      headers: Headers,
      url?: string,
    ): Promise<string | null> {
      // Header takes precedence
      let raw = headers.get(CLIENT_ID_HEADER);

      // Fallback: read from URL query string if header absent
      if (!raw && url) {
        try {
          raw = new URL(url).searchParams.get(CLIENT_ID_QUERY_PARAM);
        } catch {
          // Malformed URL — treat as no client_id supplied
          raw = null;
        }
      }

      if (!raw) return null;

      const candidate = raw.trim();
      if (!UUID_RE.test(candidate)) {
        throw new AppError("FORBIDDEN", "Invalid client_id");
      }

      const { rows } = await alwrityPool.query<{ id: string }>(
        "SELECT id FROM clients WHERE id = $1 AND is_archived = false LIMIT 1",
        [candidate],
      );

      if (rows.length === 0) {
        throw new AppError("FORBIDDEN", "Unknown or archived client_id");
      }

      return rows[0].id;
    }
    ```
