# Phase 30.5 Plan 03: Priority Scoring Algorithm Summary

**One-liner:** Weighted priority scoring (DA/traffic/opportunities/recency) auto-computed after analysis with sortable prospect list.

## Implementation

### 1. Schema Update (prospect-schema.ts)
- Added `priorityScore` column as `real` type (nullable)
- Added index `ix_prospects_priority` for efficient sorting

### 2. Priority Score Calculator (calculatePriorityScore.ts)
New utility implementing the weighted formula:
```
priorityScore = (
  (domainAuthority / 100) * 0.20 +
  min(organicTraffic / 10000, 1) * 0.15 +
  min(opportunityCount / 50, 1) * 0.25 +
  (avgOpportunityScore / 100) * 0.25 +
  recencyBonus * 0.15
) * 100
```

Recency bonus:
- Full bonus (1.0) for analyses within 7 days
- Linear decay to 0 from day 7 to day 30
- Zero bonus after 30 days

### 3. Processor Integration (prospect-analysis-processor.ts)
- Added Step 7: Calculate priority score after analysis completes
- Updates prospect record with computed score
- Logs priority score for observability

### 4. Service Query Update (ProspectService.ts)
- Added `sortBy` option to `findByWorkspace()`: "priority" | "created"
- Priority sort: `desc(priorityScore), desc(createdAt)` (nulls sort last)
- Default remains "created" for backward compatibility
