# Plan 54-01: Schema + Provider Abstraction

## Goal
Create database schema for multi-provider payment settings and implement the PaymentProvider interface with factory pattern.

## Tasks

### 1. Database Schema (2h)
- [ ] Create Drizzle schema `workspace-payment-settings-schema.ts`
  - `workspace_payment_settings` table with encrypted credential columns
  - Indexes on workspace_id
- [ ] Extend `invoice-schema.ts`
  - Add `payment_provider` column (stripe | revolut)
  - Add `revolut_order_id` column
  - Add `revolut_checkout_url` column
  - Add index on `revolut_order_id`
- [ ] Generate and apply migration

### 2. Credential Encryption Utilities (1.5h)
- [ ] Create `src/server/lib/encryption.ts`
  - `encryptCredential(plaintext: string): string`
  - `decryptCredential(ciphertext: string): string`
  - AES-256-GCM with `PAYMENT_ENCRYPTION_KEY` env var
- [ ] Unit tests for encryption roundtrip

### 3. PaymentProvider Interface (1.5h)
- [ ] Create `src/server/features/payments/types.ts`
  ```typescript
  interface PaymentSession {
    provider: 'stripe' | 'revolut';
    externalId: string;
    paymentUrl: string;
    token?: string;
  }
  
  interface WebhookEvent {
    type: string;
    orderId: string;
    data: unknown;
  }
  
  interface PaymentProvider {
    createPaymentSession(invoice: InvoiceSelect): Promise<PaymentSession>;
    verifyWebhook(rawBody: Buffer, headers: Headers): WebhookEvent;
    getPaymentStatus(externalId: string): Promise<PaymentStatus>;
  }
  ```

### 4. PaymentProviderFactory (1.5h)
- [ ] Create `src/server/features/payments/PaymentProviderFactory.ts`
  - `getProvider(workspaceId, preferredProvider?)` returns correct provider
  - Decrypts credentials from workspace settings
  - Falls back to default provider if preferred not enabled
- [ ] Create `WorkspacePaymentSettingsRepository.ts`
  - `getByWorkspaceId(workspaceId): WorkspacePaymentSettings | null`
  - `upsert(settings): WorkspacePaymentSettings`

### 5. Refactor StripeService → StripeProvider (1.5h)
- [ ] Create `src/server/features/payments/providers/StripeProvider.ts`
  - Implements `PaymentProvider` interface
  - Move existing StripeService logic into class methods
- [ ] Keep `StripeService.ts` as thin wrapper for backwards compatibility
