# Plan 54-04: Payment Settings UI + Client Choice

## Goal
Create workspace payment settings UI and client-facing payment method selection.

## Tasks

### 1. Payment Settings API Routes (2h)
- [ ] `GET /api/settings/payments` — Get workspace payment settings
- [ ] `PUT /api/settings/payments` — Update settings
- [ ] `POST /api/settings/payments/connect/stripe` — Initiate Stripe Connect
- [ ] `POST /api/settings/payments/connect/revolut` — Store Revolut credentials
- [ ] `DELETE /api/settings/payments/disconnect/:provider` — Remove credentials

### 2. Payment Settings Page (3h)
- [ ] Create `apps/web/src/app/(shell)/settings/payments/page.tsx`
- [ ] Components:
  ```tsx
  <PaymentSettingsPage>
    <ProviderSection>
      <ProviderCard provider="stripe" ... />
      <ProviderCard provider="revolut" ... />
    </ProviderSection>
    
    <DefaultProviderSelect 
      options={enabledProviders}
      value={defaultProvider}
      onChange={...}
    />
    
    <Switch 
      label="Allow clients to choose payment method"
      checked={allowClientChoice}
    />
    
    <PaymentTermsInput 
      value={paymentTermsDays}
      onChange={...}
    />
  </PaymentSettingsPage>
  ```

- [ ] ProviderCard component:
  - Logo + provider name
  - Features list (Cards, Apple Pay, etc.)
  - Connection status badge
  - Connect/Disconnect button
  - Credential input modal for Revolut

### 3. Revolut Credential Input Modal (1.5h)
- [ ] Create `RevolutConnectModal.tsx`
  - Secret Key input (masked)
  - Public Key input
  - Merchant ID input
  - Webhook Secret input
  - "Test Connection" button
  - Save only if test succeeds

### 4. Client Payment Page Updates (2h)
- [ ] Update invoice payment page to show provider choice
- [ ] `PaymentMethodSelector` component:
  ```tsx
  interface Props {
    providers: ('stripe' | 'revolut')[];
    selected: string;
    onSelect: (provider: string) => void;
  }
  
  // Visual cards with provider logos
  // Highlight selected provider
  // Show supported payment methods per provider
  ```

- [ ] Conditional rendering:
  - Single provider → direct to that provider's checkout
  - Multiple providers + allowClientChoice → show selector
  - Multiple providers + !allowClientChoice → use default

### 5. Invoice Email Updates (1h)
- [ ] Update invoice email template
  - Include payment URL (stripe or revolut)
  - Show available payment methods
  - Provider-specific instructions if needed

### 6. v6 Design Compliance (1h)
- [ ] Apply design tokens from design-system-v6.md
  - Ghost-edge shadows on cards
  - Emerald accent for connected status
  - Proper spacing and typography
- [ ] Responsive layout for mobile
