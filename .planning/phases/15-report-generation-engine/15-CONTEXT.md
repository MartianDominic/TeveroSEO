# Phase 15: Report Generation Engine - Context

**Gathered:** 2026-04-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Generate digital-first reports from analytics data with PDF export capability. Report templates as React components that render to both interactive HTML (primary) and PDF (secondary export). Template system designed for future report builder UI. Reports stored in filesystem with metadata in PostgreSQL.

</domain>

<decisions>
## Implementation Decisions

### Report Architecture (Digital-First)
- Primary format is interactive HTML (responsive, scrollable, hover states)
- PDF is a secondary export option via Puppeteer
- Template-based system with configurable sections (builder-ready)
- Same React components render to both HTML and PDF
- Shareable report links for client access (future phase)

### Report Layout & Design
- Responsive layout (not fixed A4) — works on any screen size
- Section order: Header → Summary Stats → GSC Chart → GA4 Chart → Top Queries → Footer
- Recharts with SVG output — works in both HTML and PDF, vector-crisp
- Theme-aware colors for HTML, print stylesheet for PDF export
- Each section is a standalone component (enables builder drag-drop later)

### PDF Generation Approach
- Puppeteer containerized in Docker — isolated, consistent, no system deps
- BullMQ job with dedicated worker — non-blocking, handles long renders, retry support
- Cache generated PDFs by content hash — regenerate only when data changes
- 60 second timeout with graceful failure and clear error messaging

### Report Storage & Access
- Local filesystem `/data/reports/` — volume-mounted in Docker, simple
- File naming: `{client_id}/{YYYY-MM-DD}_{report_type}.pdf`
- Access control via Clerk session + client ownership check (consistent with existing auth)
- Keep all reports indefinitely — agency needs historical access, storage is cheap

### Report Data & Content
- Default date range: Last 30 days (standard SEO reporting period)
- Comparison: Previous 30 days for trend indicators
- Top queries: 20 queries displayed (balance of insights vs. readability)
- GSC metrics: Clicks, Impressions, CTR, Avg Position
- GA4 metrics: Sessions, Users, Bounce Rate, Conversions

### Language & Localization
- Report UI language selectable per report (dropdown: EN, DE, LT, etc.)
- Keywords/queries shown in original language (as tracked)
- Report labels, headings, and descriptions localized to selected language
- Date/number formatting follows selected locale (e.g., "19 Apr" vs "Apr 19", "1.234" vs "1,234")
- Default language: English (can be overridden per client in settings)

### Claude's Discretion
- Specific chart styling details (line thickness, colors, tooltips)
- Loading states and skeleton components
- Error boundary implementation
- Report preview thumbnail generation (if needed)

</decisions>

<specifics>
## Specific Ideas

### Template System Design
```typescript
interface ReportTemplate {
  id: string;
  name: string;
  sections: ReportSection[];
  branding?: BrandingConfig;
}

interface ReportSection {
  type: 'summary_stats' | 'gsc_chart' | 'ga4_chart' | 'queries_table' | 'custom';
  config: Record<string, unknown>;
  order: number;
}
```

### Default Report Template
The "Monthly SEO Report" template includes:
1. Header (client name, date range, generated date)
2. Summary Stats (4 cards: clicks, impressions, sessions, conversions)
3. GSC Performance Chart (30-day trend)
4. GA4 Traffic Chart (30-day trend)
5. Top Queries Table (20 queries with position changes)
6. Footer (agency branding, generated by Tevero)
