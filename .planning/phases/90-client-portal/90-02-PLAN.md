Create GET /api/portal/dashboard/:clientId route.

**Request:** Token in Authorization header (Bearer token) or query param `?token=xxx`

**Response structure (per PRD dashboard spec):**
```typescript
{
  success: true,
  data: {
    metrics: {
      clicks: number,
      impressions: number,
      avgPosition: number,
      top10Count: number,
      deltas: {
        clicks: number,      // % change vs previous period
        impressions: number,
        avgPosition: number, // negative is improvement
        top10Count: number
      }
    },
    recentWins: Array<{
      keyword: string,
      position: number,
      previousPosition: number,
      date: string
    }>,
    needsAttention: Array<{
      keyword: string,
      position: number,
      previousPosition: number,
      dropAmount: number
    }>,
    lastUpdated: string  // ISO timestamp of most recent GSC data
  }
}
```

**Token validation:** Extract token from header/query, validate via portalTokenService.validateToken(), verify clientId matches.

**Error responses:**
- 401: Invalid/expired/revoked token
- 404: No GSC data for client
- 500: Internal error

Follow scope.$contractId.ts pattern exactly. Use DashboardService from 90-01.
