# Plan 73-04: RAG-Classification Pipeline Integration

## Goal

Wire LightRAG context retrieval into the ClassificationPipeline so keyword classification uses knowledge graph context for better taxonomy selection.

## Research Prescription

From cpu-only-rag-graph.md:

Pipeline architecture for e-commerce keyword classification: crawl, clean, ingest into LightRAG, query in hybrid mode, feed retrieved context into a strict-JSON GPT-4o-mini classification prompt that outputs primary_category, keywords, confidence.

## Current State

- LightRAG: Fully implemented with hybrid query mode
- ClassificationPipeline: Works independently, calls LLM without RAG context
- Location: open-seo-main/src/server/features/keywords/classification/ClassificationPipeline.ts

Current flow: Page content to LLM prompt to Classification result

Required flow: Page content to LightRAG hybrid query to Retrieved context to LLM prompt with context to Classification result

## Implementation

### Task 1: LightRAG Context Retriever (2h)

Create open-seo-main/src/server/features/keywords/classification/rag-context.ts with:

- getClassificationContext(tenantId, pageContent) function
- Uses lightragService.query with mode: hybrid, onlyNeedContext: true
- Parses entities, relationships, relevantCategories from result
- Returns RAGContext object with entities, relationships, relevantCategories, confidence

### Task 2: Integrate into ClassificationPipeline (2.5h)

Modify ClassificationPipeline.ts:

1. Import getClassificationContext
2. Call getClassificationContext before LLM
3. Build enhanced prompt with RAG context section showing entities and suggested categories
4. Include ragContextUsed and ragConfidence in result

### Task 3: Handle Fallback for No Context (0.5h)

Wrap RAG context retrieval in try-catch. Log warning but proceed without context if RAG fails.
RAG is enhancement, not requirement.

### Task 4: Tests (1h)

Create rag-context.test.ts:
- Test that retrieves relevant entities from LightRAG
- Test that returns low confidence when no entities found
- Test that ClassificationPipeline uses RAG context in prompt
- Test that falls back gracefully when RAG fails

## Verification

1. RAG context retrieval works and returns entities
2. Classification prompt includes RAG context when available
3. Classification falls back gracefully when RAG fails
4. Manual evaluation: Classifications with RAG context are more accurate

## Expected Impact

- Better category selection (uses knowledge graph taxonomy)
- Higher confidence scores (grounded in existing data)
- Reduced hallucination (LLM sees what entities exist)
- Consistent keyword vocabulary (uses terms from knowledge graph)
