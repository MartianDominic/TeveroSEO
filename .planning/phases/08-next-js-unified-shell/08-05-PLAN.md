Every page file is a **client component** (starts with `"use client";`). Server component wrappers are only added if the page needs server-only data fetching (none here — all use Zustand + fetch on mount).

Dynamic routes syntax:
- `apps/web/src/app/(shell)/clients/[clientId]/page.tsx` exports:
```tsx
export default function Page({ params }: { params: Promise<{ clientId: string }> }) { ... }
```

For client components, read params via `useParams()` from `next/navigation` since async params can't be awaited in a client component. Pattern:
```tsx
"use client";
import { useParams } from "next/navigation";
export default function Page() {
  const { clientId } = useParams<{ clientId: string }>();
  // ...
}
```

Alternative: wrap client content in a thin server component that awaits params and passes to client. Use whichever is cleaner for each page.

Every route handler:
```ts
import { NextResponse } from "next/server";
import { getFastApi, FastApiError } from "@/lib/server-fetch";
export const runtime = "nodejs";
export const dynamic = "force-dynamic";

export async function GET(req: Request, { params }: { params: Promise<{ clientId: string }> }) {
  const { clientId } = await params;
  const url = new URL(req.url);
  const qs = url.search; // forward query string
  try {
    const data = await getFastApi(`/api/analytics/${clientId}${qs}`);
    return NextResponse.json(data);
  } catch (err) {
    if (err instanceof FastApiError) {
      return NextResponse.json(err.body ?? { error: err.message }, { status: err.status });
    }
    return NextResponse.json({ error: "Internal error" }, { status: 500 });
  }
}
```

Create `apps/web/src/app/(shell)/clients/[clientId]/layout.tsx` — ensures that when the URL has a clientId, the cookie is synced. This is a client layout because Zustand can't be called server-side. Pattern: reads `clientId` from params, calls `setActiveClient(clientId)` on mount if different from `activeClientId`. This wires URL param → clientStore → cookie (via the persist middleware), which then lets server components and downstream server actions read the correct client via the cookie helper.

```tsx
"use client";
import { useEffect } from "react";
import { useParams } from "next/navigation";
import { useClientStore } from "@/stores";

export default function ClientIdLayout({ children }: { children: React.ReactNode }) {
  const { clientId } = useParams<{ clientId: string }>();
  const activeClientId = useClientStore((s) => s.activeClientId);
  const setActiveClient = useClientStore((s) => s.setActiveClient);

  useEffect(() => {
    if (clientId && clientId !== activeClientId) {
      setActiveClient(clientId);
    }
  }, [clientId, activeClientId, setActiveClient]);

  return <>{children}</>;
}
```
