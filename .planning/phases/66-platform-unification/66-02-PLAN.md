<!-- Pixel event payload from browser -->
```typescript
interface PixelEvent {
  siteId: string;
  event: 'pageview' | 'scroll' | 'click' | 'cwv' | 'ping';
  data: {
    url?: string;
    referrer?: string;
    userAgent?: string;
    // CWV metrics
    lcp?: number;
    cls?: number;
    inp?: number;
    // Scroll depth
    depth?: number;
    // Click data
    selector?: string;
    href?: string;
  };
  timestamp: number;
  sessionId: string;
}

interface VerificationStatus {
  status: 'pending' | 'detected' | 'verified' | 'error';
  firstPing?: Date;
  lastPing?: Date;
  pingCount: number;
  location?: string; // GeoIP for "we detected a visitor from..."
}
```

  <action>
Create `PixelCollectorService` that:

1. **processEvent(event: PixelEvent)**: Main entry point
   - Validate siteId exists in pixelInstallations
   - Return early with error if not found
   - Update installation: lastPingAt, increment pingCount
   - If firstPingAt is null, set it and change status to 'detected'
   - Dispatch to appropriate handler based on event.event type

2. **handlePageview(event)**: Update daily aggregates
   - Upsert into pixelAnalyticsDaily for today's date
   - Increment pageviews count
   - Track unique visitors via sessionId (store in Redis set per day)

3. **handleCwv(event)**: Aggregate Core Web Vitals
   - Store raw values in Redis sorted set
   - Calculate p75 on hourly basis via background job
   - Update pixelAnalyticsDaily.lcpP75, clsP75, inpP75

4. **handleScroll/handleClick**: Track engagement
   - Store in Redis for real-time, batch to DB hourly

Use Redis for real-time counters, batch writes to Postgres every 5 minutes to minimize DB load.
