Create `open-seo-main/src/db/pixel-schema.ts` with 4 tables per DESIGN.md Section 10:

1. **pixelInstallations** table:
   - id (text PK with nanoid)
   - workspaceId (FK to organization.id)
   - siteId (text, unique per workspace - generated UUID for pixel data-site attr)
   - status: 'pending' | 'detected' | 'verified' | 'error'
   - firstPingAt, lastPingAt, pingCount (detection tracking)
   - features (JSONB with analytics, cwv, metaInjection, schemaInjection, linkInjection, abTesting booleans)
   - allowedOrigins (text[] for domain whitelist)
   - createdAt, updatedAt timestamps

2. **pixelDomChanges** table:
   - id (text PK)
   - installationId (FK to pixelInstallations.id)
   - changeType: 'meta_title' | 'meta_description' | 'canonical' | 'schema' | 'internal_link' | 'content'
   - targetSelector (CSS selector)
   - targetUrl (page-specific changes)
   - oldValue, newValue (text)
   - status: 'pending' | 'approved' | 'rejected' | 'live' | 'rolled_back'
   - approvedBy (FK to user), approvedAt, deployedAt timestamps
   - createdAt timestamp

3. **pixelAnalyticsDaily** table:
   - id (text PK)
   - installationId (FK)
   - date (DATE, unique with installationId)
   - pageviews, sessions, uniqueVisitors (integer)
   - avgTimeOnPage, bounceRate (numeric)
   - lcpP75, clsP75, inpP75 (CWV aggregates, numeric)
   - topPages (JSONB)
   - Unique constraint on (installationId, date)

4. **developerHandoffs** table:
   - id (text PK)
   - siteId (text, references pixel installation conceptually)
   - developerEmail, developerName (text)
   - status: 'sent' | 'opened' | 'completed' | 'expired'
   - magicLinkToken (unique text)
   - magicLinkExpiresAt (timestamp)
   - sentAt, openedAt, completedAt timestamps
   - reminderCount, lastReminderAt

Export all tables, relations, and inferred types. Add proper indexes on workspaceId, status, date columns.

Update `schema.ts` barrel export to include pixel-schema.ts exports.

```typescript
// Service for generating TeveroPixel scripts
export class PixelScriptService {
  // Get or create pixel installation for a workspace
  async getOrCreateInstallation(workspaceId: string, domain: string): Promise<PixelInstallation>;
  
  // Get installation by siteId
  async getInstallationBySiteId(siteId: string): Promise<PixelInstallation | null>;
  
  // Generate the pixel script snippet for embedding
  generatePixelScript(siteId: string): string;
  
  // Generate the full pixel loader (t.js content)
  generatePixelLoader(): string;
}
```

The pixel script snippet should be:
```html
<script async src="https://pixel.tevero.io/t.js" data-site="SITE_ID"></script>
```

The loader (t.js) content should:
1. Be under 5KB uncompressed
2. Fetch config from API endpoint
3. Initialize analytics module (pageview, scroll, clicks, CWV)
4. Initialize DOM mutation module (if approved changes exist)
5. Use Navigator.sendBeacon for reliable data collection
6. Handle SPA navigation with MutationObserver

Create comprehensive tests covering all behaviors.

Create `index.ts` barrel export.
