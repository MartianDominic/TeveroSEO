interface PlatformDetectionResult {

Create `PlatformDetectorService` with detection logic:

**Detection methods (in order of reliability):**

1. **Subdomain patterns** (100% confidence):
   - `*.myshopify.com` -> shopify
   - `*.wordpress.com` -> wordpress_com
   - `*.wixsite.com`, `*.wix.com` -> wix
   - `*.squarespace.com` -> squarespace
   - `*.webflow.io` -> webflow
   - `*.ghost.io` -> ghost
   - `*.mybigcommerce.com` -> bigcommerce

2. **HTML signatures** (fetch homepage, 90-95% confidence):
   - `<meta name="generator" content="WordPress` -> wordpress_self_hosted
   - `<meta name="generator" content="Shopify"` -> shopify
   - `Squarespace.TemplateConfig` in HTML -> squarespace
   - `__WEBFLOW_CONTEXT__` in HTML -> webflow
   - `__wix_data__` or `wixStatic` in HTML -> wix
   - `Powered by Ghost` or ghost meta -> ghost
   - `WooCommerce` in HTML -> woocommerce
   - `Magento` or `MAGE_` in HTML -> magento

3. **Response headers** (80% confidence):
   - `X-Powered-By: Shopify`
   - `X-Wix-*` headers
   - Server headers for specific platforms

4. **GTM detection** (as enhancement, not primary platform):
   - Check for `gtm.js` or `GTM-` in HTML
   - Set `features: ['gtm_enabled']`

Use existing tiered crawler from Phase 61 to fetch HTML efficiently.
