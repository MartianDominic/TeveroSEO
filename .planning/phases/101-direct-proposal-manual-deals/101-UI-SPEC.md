## Interaction Patterns

### Quick Capture Modal (D-01)

**Trigger:** Cmd+K -> "New deal" OR Cmd+N OR FAB click

**Fields (minimum viable):**
1. Domain (auto-focus, required)
2. Contact (email or phone, required)
3. Stage dropdown (default: "New")

**Behavior:**
- Modal appears at top 20% of viewport (per v7 pattern)
- Domain field auto-completes from existing prospects
- Tab flow: Domain -> Contact -> Stage -> Create
- Creates stub immediately, opens full form option

**Motion:**
- Fade-in: 240ms ease-smooth
- Focus ring: 2px accent with 2px offset

### Payment Review Queue (D-02)

**Layout:** Card-based list, not separate inbox

**Per-payment row:**
```
[Amount]  [Payer reference]  [Suggested match]  [Confidence]  [Actions]
 EUR 350   ACME-INV-2026     Invoice #INV-042      95%        [Confirm] [Search]
```

**Actions column (hover-reveal pattern):**
- Confirm (primary, always visible if confidence >= 70%)
- Search (secondary, always visible)
- Create credit (tertiary, hover-reveal)

**Confidence indicator:**
- >= 90%: Solid accent badge
- 70-89%: Hollow warning badge
- < 70%: Hollow error badge with "Needs review"

**One-click confirm:**
- Confirm button: 160ms color transition
- Row fades out: 200ms
- Success toast: bottom-right

### Command Palette (D-03)

**Trigger:** Cmd+K globally

**Context groups (dynamic based on current view):**

When on Pipeline view:
- "New deal" (Cmd+N)
- "Quick capture" (Cmd+Shift+N)
- "Search deals..."
- "Record payment"
- "Close deal"

When on Deal detail:
- "Send proposal" (S)
- "Record payment" (P)
- "Edit" (E)
- "Add comment" (C)
- "Mark as accepted"
- "Upload signed doc"

**Styling (v6 tokens):**
- Background: `--surface` with `--shadow-modal`
- Input: 14px body, Search icon 16px
- Group headings: `.t-eyebrow`
- Items: 14px, highlight with `--accent-soft`
- Shortcuts: `.kbd` chip right-aligned

### Deal Card Actions (D-03)

**4 primary actions with shortcuts (1-4):**

Stage: New
1. Edit (E)
2. Generate proposal (G)
3. Add note (N)
4. Archive (A)

Stage: Sent
1. Mark viewed (V)
2. Send reminder (R)
3. Record call (C)
4. Archive (A)

Stage: Viewed
1. Mark accepted (M)
2. Upload signed (U)
3. Record payment (P)
4. Call (C)

Stage: Signed
1. Record payment (P)
2. Convert to client (X)
3. Add note (N)
4. Archive (A)

**Hover-reveal pattern:**
- Action bar hidden at rest
- Appears on card hover: 240ms fade-in
- Keyboard shortcut badges visible on hover

### AI Mode Selection (D-03)

**4-tier tabs (horizontal):**

| Mode | Icon | Description | Shortcut |
|------|------|-------------|----------|
| Full AI | Sparkles | AI generates everything | 1 |
| AI-Assisted | Edit | You provide key details | 2 |
| Template | FileText | Pick template, fill manually | 3 |
| Blank | Plus | Start from scratch | 4 |

**Selected state:** `--accent-soft` background, `--accent-ink` text
**Unselected:** `--surface-2` background, hover to `--surface-3`

**Visualization:** Horizontal bar per section

[Deliverables]     ██████████████████████████  8m 15s  <- most viewed

## Accessibility Requirements

### WCAG 2.1 AA Compliance

| Requirement | Implementation |
|-------------|----------------|
| Text contrast | All text >= 4.5:1 on backgrounds |
| Focus visible | 2px accent ring on all focusables |
| Touch targets | 44x44px minimum |
| Motion | Respect `prefers-reduced-motion` |
| Screen reader | ARIA labels on all actions |

### ARIA Patterns

**Quick Capture Modal:**
- `role="dialog"`
- `aria-modal="true"`
- `aria-labelledby` -> title
- Focus trap with `FocusTrap` component

**Payment Review Queue:**
- `role="table"` or `role="list"`
- Each row: `role="row"` with `aria-label` describing payment
- Confirm button: `aria-describedby` linking to confidence

**Command Palette:**
- `role="combobox"` with `aria-expanded`
- Results: `role="listbox"`
- Items: `role="option"` with `aria-selected`

**Deal Cards:**
- `role="listitem"` within pipeline `role="list"`
- Action buttons: `aria-label` describing action
- Keyboard shortcuts: `aria-keyshortcuts` attribute

## Implementation Notes

### Component Build Order

1. **QuickCaptureModal** - Unblocks deal creation flow
2. **DealCard + ProposalCardActions** - Unblocks pipeline interactions
3. **PaymentReviewQueue + PaymentMatchCard** - Unblocks reconciliation
4. **AIModeTabs** - Unblocks proposal generation
5. **ContentLibraryPanel** - Can be added incrementally
6. **SectionHeatmap** - Analytics layer, can be deferred

### v6 Token References

All components must use v6 CSS custom properties:
- `var(--surface)` not `bg-white`
- `var(--text-1)` not `text-gray-900`
- `var(--shadow-card)` not custom shadows
- `var(--space-N)` not arbitrary Tailwind spacing

### Existing Component Reuse

- Use `Card`, `CardHeader`, `CardContent`, `CardFooter` from @tevero/ui
- Use `Badge` with semantic variants
- Use `Button` with `primary`/`ghost`/`destructive` variants
- Use `Dialog` primitives, extend with v6 styling
- Use `Command` from cmdk, restyle with v6 tokens
- Use `EmptyState` component for all empty states
- Use `StepIndicator` for form wizards if needed
