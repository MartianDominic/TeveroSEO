<objective>
Implement Cheerio-based web scraper with smart link detection for prospect website analysis.

Purpose: Enable scraping of static HTML websites (80% of sites) to understand what a prospect sells, their services, and business focus -- even when they have zero keyword rankings.

Output: Scraper module in open-seo-main that fetches pages, detects business-relevant links (/products, /about, /services), and extracts structured content (title, meta, headings, body text).
</objective>

export interface ScrapedPage {
  url: string;
  statusCode: number;
  title: string | null;
  metaDescription: string | null;
  headings: { level: number; text: string }[];
  bodyText: string;
  links: string[];

  products: string | null;  // /products, /shop, /store
  about: string | null;     // /about, /about-us, /company

}

export interface ExtractedContent {

  metaDescription: string | null;
  headings: { level: number; text: string }[];
  bodyText: string;

}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scraper -> external site | Untrusted HTML from arbitrary websites |
| external site -> scraper | Response content could be malicious |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-01 | Spoofing | cheerioScraper | mitigate | User-Agent identifies as TeveroBot; respect robots.txt |
| T-27-02 | Tampering | contentExtractor | mitigate | Parse only, no eval; cheerio handles HTML safely |
| T-27-03 | Denial of Service | cheerioScraper | mitigate | 10s timeout, max 50KB body text, max 5 redirects |
| T-27-04 | Information Disclosure | linkDetector | accept | Links from public HTML pages only; no auth bypass |
| T-27-05 | Elevation of Privilege | JSON-LD parsing | mitigate | JSON.parse only, no code execution; catch parse errors |
</threat_model>
