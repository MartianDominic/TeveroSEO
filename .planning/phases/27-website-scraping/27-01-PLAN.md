DataForSEO On-Page API endpoints (workflow):
```
1. POST /v3/on_page/content_parsing/live
   Body: { url, enable_javascript: true, store_raw_html: true }
   Response: { tasks: [{ id, result }] }

2. POST /v3/on_page/raw_html
   Body: { id: taskId }
   Response: { tasks: [{ result: [{ items: [{ page_content: { html } }] }] }] }
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scraper -> DataForSEO API | Trusted third-party API with authentication |
| DataForSEO -> external site | DataForSEO handles rendering; we receive HTML |
| page-analyzer -> HTML | Untrusted HTML from arbitrary websites |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-01 | Spoofing | dataforseoScraper | mitigate | Use authenticated API with DATAFORSEO_API_KEY |
| T-27-02 | Tampering | page-analyzer | mitigate | Parse only, no eval; cheerio handles HTML safely |
| T-27-03 | Denial of Service | DataForSEO API | mitigate | API has rate limits; billing metering prevents abuse |
| T-27-04 | Information Disclosure | fetchRawHtml | accept | API key in env var, not in code; HTTPS transport |
| T-27-05 | Elevation of Privilege | page-analyzer | mitigate | JSON.parse only for structured data; no code execution |
</threat_model>
