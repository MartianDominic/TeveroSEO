# Plan 91-04: Model ID Migration

## Objective

Update all deprecated model IDs to current versions. This ensures quality, access to latest features, and prevents API deprecation issues.

## Prerequisites

- Plan 91-02 complete (prompt caching)
- Test environment available for quality validation

## Background

| Deprecated Model | Current Model | Reason |
|------------------|---------------|--------|
| `grok-2-mini` | `grok-4-1-fast-reasoning` | Grok 2 series deprecated |
| `gemini-1.5-pro` | `gemini-3.1-pro` | Gemini 1.5 deprecated |
| `gemini-2.5-flash-lite` | `gemini-3.1-flash-lite` | Gemini 2.5 deprecated |
| `claude-3-5-sonnet-*` | `claude-sonnet-4-6` | Old naming convention |

**Why This Matters:**
- Deprecated models may be removed from APIs
- Current models have better quality/cost ratios
- Prompt caching features differ by model version

## Risk Mitigation

**Why `autonomous: false`:** Model changes can affect output quality. Human review required for:
- Voice analysis samples (does it capture brand tone?)
- Translation samples (Lithuanian accuracy)
- Classification samples (correct category assignment)

## Rollback Plan

Each model change is independent. If quality issues detected:
1. Revert specific file to previous model ID
2. Add model to "needs investigation" list
3. Continue with other migrations

## Quality Test Protocol

For each model change:
1. Run 10 sample inputs with old model (baseline)
2. Run same 10 inputs with new model
3. Compare outputs manually
4. If quality equivalent or better: approve
5. If quality worse: investigate or revert
