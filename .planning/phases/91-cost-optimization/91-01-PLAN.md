# Plan 91-01: Cost Optimization Implementation

## Objective

Implement Phase 91 cost savings by eliminating wasteful scheduled jobs and extending cache TTLs to match SEO data volatility.

## Tasks

### T1: Eliminate Daily Ranking Scheduler ✅
**File:** `open-seo-main/src/server/queues/rankingQueue.ts`
- Remove daily 03:00 UTC cron that checks ALL tracked keywords
- Keep `triggerRankingCheck()` for manual refresh
- **Savings:** $100-300/month

### T2: Disable 5-Minute Alert Polling ✅
**File:** `open-seo-main/src/server/queues/alertDetectionQueue.ts`
- Remove 5-minute repeatable job
- Alerts now triggered on-demand after metrics update
- **Savings:** 288 job runs/day eliminated

### T3: Disable 5-Minute Aggregates Polling ✅
**File:** `open-seo-main/src/server/queues/portfolioAggregatesQueue.ts`
- Remove 5-minute repeatable job
- Aggregates now triggered on-demand after metrics update
- **Savings:** 288 job runs/day eliminated

### T4: Extend Backlinks TTL 6h → 24h ✅
**File:** `open-seo-main/src/server/features/backlinks/services/backlinksServiceData.ts`
- `BACKLINKS_OVERVIEW_TTL_SECONDS = 24 * 60 * 60`
- **Savings:** 75% fewer backlinks API calls

### T5: Extend Domain TTL 12h → 7 days ✅
**File:** `open-seo-main/src/server/features/domain/services/DomainService.ts`
- `DOMAIN_OVERVIEW_TTL_SECONDS = 7 * 24 * 60 * 60`
- **Savings:** 85% fewer domain API calls

### T6: Extend GSC TTL 1h → 6h ✅
**File:** `open-seo-main/src/server/services/GscBridgeService.ts`
- `CACHE_TTL_SECONDS = 6 * 60 * 60`
- **Savings:** 83% fewer Redis reads

### T7: Extend Keyword Volume TTL 7d → 30d ✅
**File:** `open-seo-main/src/server/features/keywords/config/routing.ts`
- `DATAFORSEO_LABS: 30 * 24 * 60 * 60`
- **Savings:** 75% fewer volume API calls
