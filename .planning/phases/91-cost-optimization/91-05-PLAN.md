### T2: Investigate B1 - Classification TTL Extension

**File:** `open-seo-main/src/server/features/keywords/services/ClassificationSingleflight.ts`

**Question:** Should classification cache be 30 days (current after 91-01) or 90+ days?

**Current:** `resultTTL: 2592000` (30 days after Phase 91)

**Consideration:**
- Classification IS deterministic (same input = same output)
- BUT: Category taxonomy might change
- BUT: Re-classification provides opportunity to fix errors

**Decision criteria:**
- If taxonomy stable: Extend to 90 days
- If taxonomy evolves frequently: Keep at 30 days
- Consider version key pattern: `class:v1:{hash}` allows invalidation on taxonomy change

**Output:** TTL decision + implementation if changed

### T3: Investigate B4 - Research TTL Extension

**File:** `open-seo-main/src/server/features/keywords/services/research/research.ts`

**Question:** Should keyword research results cache 7 days instead of 24h?

**Current:** `CACHE_TTL.researchResult: 86400` (24h)

**Consideration:**
- Keyword research results are stable for weeks
- BUT: User might expect "fresh" research
- BUT: Re-research wastes money for same results

**Decision criteria:**
- If research results rarely change: Extend to 7 days
- Add "Research date: {date}" to UI for transparency
- Keep manual refresh button available

**Output:** TTL decision + implementation if changed

### T4: Verify C1 - TranslationService Batching

**File:** `open-seo-main/src/server/services/translation/TranslationService.ts`

**Question:** Is current batching "real" or "fake"?

**Fake batching pattern (BAD):**
```typescript
// Makes N API calls, not 1
await Promise.all(batch.map(text => this.translate(text)));
```

**Real batching pattern (GOOD):**
```typescript
// Makes 1 API call for N texts
const prompt = texts.join('\n---\n');
const response = await gemini.generate(prompt);
```

**Investigation:** Read TranslationService code, determine pattern

**Output:** If fake batching found, add to Plan 91-03

### T6: Investigate D3 - Flash vs Pro for Articles

**Question:** Should standard articles use Gemini Flash instead of Pro?

**Cost comparison:**
- Gemini 3.1 Pro: $1.25/1M tokens
- Gemini 3.1 Flash: $0.075/1M tokens (16x cheaper!)

**Quality tradeoff:**
- Pro: Better for nuanced, brand-voice content
- Flash: Good for straightforward content

**Decision criteria:**
- If quality gap is noticeable: Keep Pro for all
- If quality gap is minimal: Flash for standard, Pro for premium
- Test with 10 sample articles

**Output:** Model tiering decision for AI-Writer

### T7: Investigate D4 - Tiered Voice Compliance

**Question:** Should voice compliance scoring use deterministic checks before LLM?

**Current:** Every compliance check calls Claude

**Proposed:** 
1. Deterministic score first (word choice, sentence length, etc.)
2. Only call LLM for borderline scores (70-85%)

**Decision criteria:**
- If most content scores <70% or >85%: Tiering saves 60-80%
- If most content is borderline: Tiering saves less

**Investigation:** Sample compliance scores to determine distribution

**Output:** Tiering decision + implementation plan if approved
