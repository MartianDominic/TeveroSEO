# Plan 91-02: Prompt Caching Implementation

## Objective

Implement prompt caching for xAI Grok and Gemini to achieve **75-90% cost reduction** on repeated system prompts. This is "free money" — same output quality, lower cost.

## Prerequisites

- Plan 91-01 complete (Wave 1 optimizations) ✅
- XAI_API_KEY configured
- GOOGLE_AI_API_KEY configured

## Background

| Provider | Feature | Discount | How It Works |
|----------|---------|----------|--------------|
| xAI Grok | Automatic caching | 75% on cached tokens | No code changes needed — just measure |
| Gemini | Context caching | 90% on cached input | Explicit CachedContent API |

**Key Insight:** xAI auto-caches by default. We just need to verify it's working and optimize prompt structure. Gemini requires explicit caching API.

## Tasks

### T1: Verify xAI Auto-Caching is Active

**File:** `open-seo-main/src/server/features/keywords/services/GrokClassifier.ts`

**Current:** No cache verification in place

**Change:** Add cache hit logging to verify auto-caching works:

```typescript
// After API call
const cached = response.usage.prompt_tokens_details?.cached_tokens || 0;
const total = response.usage.prompt_tokens;
if (cached > 0) {
  log.info('Grok cache hit', {
    cachedTokens: cached,
    totalPromptTokens: total,
    savingsPercent: Math.round((cached / total) * 100),
  });
}
```

**Verification:** Run classification batch, check logs for "Grok cache hit" entries

### T2: Optimize Grok Prompt Structure for Caching

**File:** `open-seo-main/src/server/features/keywords/classification/prompts.ts`

**Current:** May have dynamic content mixed with static

**Change:** Front-load static content, append dynamic content:

```typescript
// GOOD: Static first (cached), dynamic last (not cached)
const systemPrompt = CLASSIFICATION_SYSTEM_PROMPT; // ~350 tokens (cached)
const userPrompt = `Classify these keywords:\n${keywords.join('\n')}`; // Dynamic

// BAD: Dynamic mixed in (breaks caching)
const systemPrompt = `You are classifying for ${clientName}...`; // Never cached
```

**Verification:** Cache hit rate should be >80% after fix

### T3: Implement Gemini Context Caching for Voice Analysis

**File:** `open-seo-main/src/server/features/voice/services/VoiceAnalyzer.ts`

**Current:** Voice analysis system prompt sent fresh each call

**Change:** Use Gemini CachedContent API:

```typescript
import { CachedContent } from '@google/generative-ai';

// Create cached context (run once per session)
const voiceCache = await CachedContent.create({
  model: 'gemini-3.1-pro',
  contents: [{ role: 'user', parts: [{ text: VOICE_SYSTEM_PROMPT }] }],
  ttlSeconds: 3600, // 1 hour
});

// Use cached context for analysis
const model = genAI.getGenerativeModel({
  model: 'gemini-3.1-pro',
  cachedContent: voiceCache,
});
```

**Verification:** `result.usageMetadata.cachedContentTokenCount` should be >0
