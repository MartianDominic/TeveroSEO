# 98-09: World-Class Tool Progress Implementation

## Goal

Implement real-time tool progress tracking in SEO Chat UI. Currently the `useToolProgress` hook exists but was never wired into components. This creates a static skeleton UX instead of world-class streaming feedback during long operations (keyword analysis: 30-60s).

## Problem Statement

The `useToolProgress` hook provides:
- 4 execution states: `pending` → `streaming` → `complete` / `error`
- Partial results during streaming
- Type guards for result validation
- Error handling with messages

But ProspectContext and ToolResultCard show **generic skeletons** instead of:
- "Found 47 keywords..." during streaming
- "Analyzing groziosalon.lt..." with actual domain
- Error messages when tools fail
- Progress indicators for long operations

### Task 2: Enhance ProspectContext with Streaming States
**File:** `apps/web/src/components/seo-chat/ProspectContext.tsx`

Add real-time progress tracking:

1. Restore props interface:
```tsx
import type { ToolProgress } from '@/hooks/useToolProgress';

export interface ProspectContextProps {
  context: SessionContext | null;
  analyzing: string | null;
  toolProgress: ToolProgress[];
}
```

2. Extract tool-specific progress:
```tsx
// Get progress for each tool type
const domainProgress = toolProgress.find(t => t.toolName === 'domain_health');
const keywordProgress = toolProgress.find(t => t.toolName === 'keyword_analysis');
const feasibilityProgress = toolProgress.find(t => t.toolName === 'feasibility_check');
```

3. Show streaming state with partial results:
```tsx
// Domain Health - show domain being analyzed
{domainProgress?.state === 'streaming' && domainProgress.partialResult?.domain && (
  <p className="text-sm text-muted-foreground animate-pulse">
    Analyzing {domainProgress.partialResult.domain}...
  </p>
)}

// Keywords - show count during streaming  
{keywordProgress?.state === 'streaming' && (
  <div className="animate-pulse">
    <p className="text-sm">
      Found {(keywordProgress.partialResult as any)?.keywords?.length ?? 0} keywords...
    </p>
  </div>
)}
```

4. Show error states:
```tsx
{domainProgress?.state === 'error' && (
  <div className="flex items-center gap-2 text-destructive text-sm">
    <AlertCircle className="h-4 w-4" />
    <span>{domainProgress.error ?? 'Analysis failed'}</span>
  </div>
)}
```

### Task 3: Enhance ToolResultCard with Contextual Streaming
**File:** `apps/web/src/components/seo-chat/ToolResultCard.tsx`

1. Restore partialArgs prop:
```tsx
interface ToolResultCardProps {
  toolName: string;
  state: 'pending' | 'partial-call' | 'call' | 'result';
  result?: unknown;
  partialArgs?: Record<string, unknown>;
}
```

2. Show contextual streaming messages:
```tsx
function getStreamingMessage(toolName: string, partialArgs?: Record<string, unknown>): string {
  switch (toolName) {
    case 'domain_health':
      return partialArgs?.domain 
        ? `Analyzing ${partialArgs.domain}...`
        : 'Analyzing domain health...';
    case 'keyword_analysis':
      return partialArgs?.domain
        ? `Finding keywords for ${partialArgs.domain}...`
        : 'Analyzing keywords...';
    case 'feasibility_check':
      return partialArgs?.keyword
        ? `Checking feasibility: ${partialArgs.keyword}...`
        : 'Checking feasibility...';
    case 'generate_proposal':
      return 'Generating proposal...';
    default:
      return `Running ${toolName}...`;
  }
}
```
