# Phase 17 Plan 04: Rank Drop Alerts Integration

## Objective

Add rank drop detection to the ranking worker. Store drop events for Phase 18's alert system to consume. Add configurable drop threshold to saved keywords.

## Tasks

### Task 1: Add drop threshold to saved_keywords schema

**Files:**
- `open-seo-main/src/db/schema/seo/keywords-schema.ts`

**Changes:**
1. Add `dropAlertThreshold` integer column (default 5) — alert if rank drops by this many positions
2. Generate migration

**Commit:** `feat(17-04): add drop threshold to saved_keywords`

---

### Task 2: Create rank_drop_events table

**Files:**
- `open-seo-main/src/db/schema/seo/rank-events-schema.ts`
- `open-seo-main/src/db/schema/seo/index.ts`

**Changes:**
1. Create `rankDropEvents` table:
   - `id` (text, UUID v7)
   - `keywordId` (FK to saved_keywords)
   - `projectId` (text)
   - `clientId` (text)
   - `previousPosition` (integer)
   - `currentPosition` (integer)
   - `dropAmount` (integer)
   - `threshold` (integer) — threshold that triggered this
   - `detectedAt` (timestamp)
   - `processedAt` (timestamp, nullable) — when alert system processed it
   - `processedBy` (text, nullable) — "alert_worker" or manual
2. Add index on (clientId, processedAt) for efficient queue scanning
3. Export from index.ts
4. Generate migration

**Commit:** `feat(17-04): add rank_drop_events table`

---

### Task 3: Add drop detection to ranking worker

**Files:**
- `open-seo-main/src/workers/ranking-processor.ts`
- `open-seo-main/src/services/rank-events.ts`

**Changes:**
1. Create `rank-events.ts` service:
   - `recordDropEvent(data)` — insert into rank_drop_events
   - `getUnprocessedDrops(clientId)` — for Phase 18 alert worker
2. Update ranking-processor.ts:
   - After storing new ranking, compare with previous
   - If drop > threshold, call `recordDropEvent`
   - Log drop detection for observability

**Commit:** `feat(17-04): add drop detection to ranking worker`

---

### Task 4: Add drop events API endpoint

**Files:**
- `open-seo-main/src/routes/api/clients/$clientId/drop-events.ts`

**Changes:**
1. GET endpoint returning unprocessed drop events for a client
2. Used by Phase 18 alert system

**Commit:** `feat(17-04): add drop events API`
