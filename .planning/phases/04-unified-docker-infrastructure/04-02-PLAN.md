<objective>
Create `open-seo-main/Dockerfile.vps` — a multi-stage Node.js 22 alpine build that produces a runtime image used by both the `open-seo` (HTTP server) and `open-seo-worker` (BullMQ worker) services in `docker-compose.vps.yml`. Also create `.dockerignore` to keep the build context small.

Purpose: DOCKER-01 requires a multi-stage `Dockerfile.vps` with exec-form CMD so SIGTERM reaches the node process (shell-form wraps it in `sh -c` and loses signals, breaking the graceful shutdown path already built in Phase 3 at `open-seo-main/src/server.ts:43-48`). Worker and HTTP server share the same image because both run the same built artifact `.output/server/index.mjs` — the worker variant is achieved in compose via `command:` override, not a separate Dockerfile.

Output: `open-seo-main/Dockerfile.vps` + `open-seo-main/.dockerignore`. Image builds successfully and can be introspected to confirm correct entrypoint, CMD form, user, and file layout.

  <action>
    Create `open-seo-main/Dockerfile.vps` with exactly this content:

    ```dockerfile
    # syntax=docker/dockerfile:1.7

    # ---- Stage 1: builder ----
    FROM node:22-alpine AS builder

    ENV PNPM_HOME=/pnpm
    ENV PATH=$PNPM_HOME:$PATH
    ENV NODE_ENV=production

    # Alpine needs these for some native deps during build
    RUN apk add --no-cache libc6-compat python3 make g++ \
        && corepack enable

    WORKDIR /app

    # Install deps (cached layer) — include dev deps so tsc + vite are available
    COPY package.json pnpm-lock.yaml ./
    RUN pnpm install --frozen-lockfile

    # Copy source and build
    COPY . .
    # Build produces .output/server/index.mjs (Nitro) + .output/public/ (static assets)
    RUN pnpm run build

    # Prune dev deps for runtime copy
    RUN pnpm prune --prod

    # ---- Stage 2: runtime ----
    FROM node:22-alpine AS runtime

    ENV NODE_ENV=production
    ENV PORT=3001

    # Tini for proper PID 1 signal handling (belt-and-suspenders with exec-form CMD)
    RUN apk add --no-cache tini \
        && addgroup -S nodejs -g 1001 \
        && adduser -S nodejs -u 1001 -G nodejs

    WORKDIR /app

    # Copy only what's needed to run: built output, pruned prod node_modules, package.json
    COPY --from=builder --chown=nodejs:nodejs /app/.output ./.output
    COPY --from=builder --chown=nodejs:nodejs /app/node_modules ./node_modules
    COPY --from=builder --chown=nodejs:nodejs /app/package.json ./package.json
    COPY --from=builder --chown=nodejs:nodejs /app/drizzle ./drizzle

    USER nodejs

    EXPOSE 3001

    # Tini forwards signals; CMD must be exec-form so node is PID 2 and receives SIGTERM.
    # HTTP server mode (default). Worker-only mode is achieved via compose `command:` override
    # in open-seo-worker service (docker-compose.vps.yml).
    ENTRYPOINT ["/sbin/tini", "--"]
    CMD ["node", ".output/server/index.mjs"]
    ```

    Key points enforced by this Dockerfile:
    - **Multi-stage** (builder → runtime): runtime image contains no source, no dev deps, no pnpm store
    - **node:22-alpine** (DOCKER-01 spec; smaller than node:22 used in Dockerfile.selfhost)
    - **Exec-form CMD** `["node", ".output/server/index.mjs"]` — DOCKER-01 mandates this so SIGTERM reaches the node process registered in `src/server.ts:43`
    - **tini entrypoint** — belt-and-suspenders for PID 1 reaping, standard production Node.js pattern
    - **Non-root user** (uid 1001) — security hardening
    - **drizzle/ folder copied** — needed for `drizzle-kit migrate` which is run as a separate `docker compose run --rm` step per 04-CONTEXT.md

    After writing, build locally to verify:

    docker build -f Dockerfile.vps -t open-seo:test .
