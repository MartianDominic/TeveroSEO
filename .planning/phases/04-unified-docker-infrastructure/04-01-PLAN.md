<objective>
Expose a lightweight `/healthz` endpoint on the open-seo HTTP server so the Docker healthcheck (`curl -f http://localhost:3001/healthz`) can reliably determine liveness. This is required by DOCKER-07 and is consumed by the `open-seo` service's healthcheck in `docker-compose.vps.yml` (Plan 04-04).

Purpose: Containers without a healthcheck show "starting" indefinitely and dependent services (nginx) cannot know when upstream is ready. A tiny no-dependency route returns OK without hitting DB or Redis, so transient DB/Redis blips do not mark the container unhealthy and trigger restarts.

Output: `open-seo-main/src/routes/api/healthz.ts` — a TanStack Start file route exporting GET that returns `{ status: "ok" }`.
</objective>
